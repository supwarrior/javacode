22 OM_WAFER_LEVEL_PASSCOUNT_CONTROL OMS 增加备注到代码 The env var enables to accumulate passed process count by wafer, and it enables rework tx to check the passed process count by wafer. When lot moves to a process on a process flow using tx (lot start, move out, pass through, skip, branch, rework, lot plan chg etc), the passed process count is accumulated by lot and it is stored by main process flow and ope no. And, when lot is reworked, the passed process count for the responsible operation is checked whether it is equal or less than the max process count defined in MDS.

When this env is set as 1 or 2, the passed process count is accumulated by wafer and rework transaction checks the passed process count of all wafers included in the lot.

 

0: Rework transaction checks the passed process count accumulated by Lot. (default)

1: Rework transaction checks the passed process count accumulated by wafer. The passed process count is accumulated only for the responsible process of rework process.

2: Rework transaction checks the passed process count accumulated by Wafer. The passed process count is accumulated for all processes.
