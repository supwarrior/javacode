1 OM_EDC_CHK_LEVEL 添加备注-代码 This parameter specifies collected data value check level.
“0” (Default): Data type verification is performed. Blank data value is checked.
“1”: In addition to level 0, asterisk data is checked according to DC Spec items and processed wafers. For any required item defined in DC Spec, it should have at least one valid data value present and must not be asterisk.
“2”: In addition to level 0, asterisk data is checked according to DC Spec items and processed wafers. For any required item defined in DC Spec, all items (values) defined in DCDef for processed wafer should present and must not be asterisk.
2 OM_EDC_ASTERISK_VALUE_CHK_TIME 添加备注-代码 This env specifies edc value check timing for asterisk data (*1). This env will take effect when OM_EDC_CHK_LEVEL is set to “1” or “2”.

“0” (Default): Collected data value check for asterisk is performed before or inside move out (*2).
“1”: Collected data value check for asterisk is performed as post process after move out (*3).

Note:
*1: Collected data type verification and blank data value check are always performed before or as a part of operation completion.
*2: Since collected data spec check from UI is supported only before move out,  edc value check for asterisk is skipped when claiming data spec check from UI if OM_EDC_ASTERISK_VALUE_CHK_TIME is set to “1”.
*3: When OM_EDC_ASTERISK_VALUE_CHK_TIME is set to 1, system should support lot post process operation mode (OM_PP_FOR_LOT_MOVEOUT_FLAG should be 1).

21 OM_PCS_MAX_EXECUTION_IN_CHAIN 添加备注-代码 Specify count to execute PCS in chain. When 0 is set, PCS is not called. Default value is 1. E.g. of chain execution - we have process flow with ope no 100, 200, 300 and 400. Some of the steps are attached with PCS, which will skip the lot to other operation.

   |

   O 100

   |

   O 200   PCS skip to 300

   |

   O 300   PCS skip to 400

   |

   O 400   PCS skip to 200

   |

The above scenario will turn to infinite loop. 

Using this env, PCS execution is controlled.

23 OM_APC_DATA_REQUEST_MAX_COUNT 添加备注-代码 Specify maximum number of record count to be returned from APC Entity Value Inquiry.
The default value is 1,000. e.g.
The APC sends an enquiry to OMS to get the Machine Recipe List. In OMS server, there are over 1,000,000 recipes exist. If there is no limitation on returned record count, a huge memory area will be used by OMS server and the returned result is going to be huge. OM_APC_DATA_REQUEST_MAX_COUNT will restrict the number of returned record count
